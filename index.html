Git Hub

O GitHub é uma plataforma de hospedagem de código fonte baseada em Git que facilita o gerenciamento de projetos de software e colaboração entre desenvolvedores.


Repositórios

Repositórios são locais onde seu projeto é armazenado, ele possui todos os arquivos de um projeto (commits,branches tags e metadados).

Git e Controle de Versão

O GitHub utiliza o sistema de controle de versão distribuído Git para gerenciar os arquivos dentro de cada repositório. O Git permite que várias pessoas trabalhem simultaneamente no mesmo projeto, gerenciando suas próprias cópias locais do repositório.

Colaboração e Pull Requests

Pull Request: É um mecanismo no GitHub para propor alterações no código. Visando facilitar a revisão do código, a discussão de ideias e a garantia de que as mudanças não quebrem o código existente.

Issues e Gestão de Projetos

Issues: São usadas para acompanhar tarefas, bugs, solicitações de funcionalidades e outras melhorias no projeto. Sendo que cada issue pode ser atribuída por um colaborador,comentada, marcada com etiquetas (labels), e fechada quando resolvida. Gestão de Projetos:O GitHub oferece ferramentas para gerenciar projetos e acompanhar o progresso de cada item.

Integração Contínua e Automação

GitHub Actions: Permite configurar fluxos de trabalho automatizados, como testes automatizados, compilação de código, deploy automático, entre outros, diretamente no GitHub.

Segurança e Colaboração

Controle de Acesso: Repositórios podem ser públicos (acessíveis a todos) ou privados (acesso restrito a colaboradores específicos). É possível gerenciar permissões para cada membro da equipe. Revisão de Código: A funcionalidade de revisão de código do GitHub permite que os membros da equipe revisem as alterações propostas antes da mesclagem, garantindo a qualidade do código.

Social e Comunidade

Perfil de Usuário: Cada usuário e organização no GitHub possui um perfil público ou privado que lista seus repositórios, contribuições para projetos open source, e outros detalhes.


Sistema de controle de versão GIT

O Git é um sistema de controle de versão distribuído amplamente utilizado para o de código fonte e acompanhamento de alterações em projetos de software.

Funcionamento Básico do Git

O Git armazena todas as alterações feitas nos arquivos de um projeto ao longo do tempo. Cada mudança é registrada como um "commit", que inclui um instantâneo dos arquivos naquele momento, junto com informações como autor, mensagem de commit e um identificador único (hash). Cada desenvolvedor tem uma cópia completa do repositório Git em seu computador. Isso permite que trabalhem offline, realizem commits localmente e depois sincronizem suas mudanças com outros repositórios remotos, como o GitHub.

Ramos (Branches)

Desenvolvimento Paralelo: Os ramos permitem que várias linhas de desenvolvimento ocorram simultaneamente. Integração e Fusão: Após concluir o trabalho em um ramo, é possível fundir (merge) as alterações de volta ao ramo principal ou a outro ramo.

Área de Staging (Index)

Preparação de Commits: Antes de confirmar mudanças com um commit, os arquivos são adicionados à área de staging. Isso permite revisar e selecionar quais mudanças devem ser incluídas no próximo commit.

Sincronização

Push e Pull: git push envia alterações locais para o repositório remoto, enquanto git pull atualiza o repositório local com mudanças feitas no repositório remoto.

Integração Contínua e Automação

GitHub Actions e GitLab CI: Ferramentas de integração contínua (CI) permitem automatizar testes de código, build de aplicativos e deploy automático diretamente a partir do Git.

Ramificação e Versão

Tagging: Marcadores (tags) são usados para marcar pontos específicos na história do repositório, como versões de software estáveis ou marcos importantes.


Comandos GIT

Git Init

  • Comando utilizado para inicializar um novo repositório. Ele cria uma nova instância de um repositório Git em um diretório existente ou novo.

Git Clone

  • Git clone é o comando usado para criar uma cópia exata de um repositório Git.

Git Branch

  • Ele permite criar, listar, renomear e excluir branches .

Git Checkout

  • Usado principalmente para trocar de branch ou restaurar arquivos em um repositório.

Git Status

  • É o comando que exibe o estado atual da árvore de trabalho do Git.

Git Diff

  • Compara o conteúdo de arquivos no repositório em diferentes estados.

Git Add

  • É o comando que adiciona mudanças de arquivos à área de stage (índice).

Git Commit

  • Git commit cria um snapshot permanente das alterações preparadas no índice, com uma mensagem de commit que descreve essas mudanças .

Git Push

  • É usado para enviar commits locais para um repositório remoto.

Git Pull

  • É usado para atualizar o repositório local com as mudanças do repositório remoto.

Git Revert

  • Cria um novo commit que desfaz as alterações introduzidas por um commit anterior.

Git Merge

  • É usado para mesclar alterações de um branch para outro branch.

Git Stash

  • É utilizado para armazenar temporariamente mudanças não commitadas em um espaço de trabalho Git.

Git RM

  • É utilizado para remover arquivos ou diretórios do controle de versão do Git.

Git Config

  • É utilizado para configurar variáveis de configuração do Git em diferentes níveis: sistema, usuário e repositório.

Git Reset

  • É usado para desfazer alterações e redefinir o estado do repositório Git.

Git Remote

  • É usado para listar, adicionar e gerenciar conexões com repositórios remotos.

Git Fetch

  • É usado para baixar commits, arquivos e referências de um repositório remoto para o seu repositório local.

Git Show

  • É utilizado para exibir informações detalhadas sobre um objeto Git específico.

Git Help

  • É utilizado para obter informações de ajuda detalhadas sobre comandos Git específicos.

Git Rebase

  • É usado para reorganizar o histórico de commits em um repositório.

Git Blame

  • É um comando usado para determinar qual commit e autor foram responsáveis por cada linha de um arquivo específico.

Git Tag

  • É utilizado para gerenciar tags, que são marcadores usados para marcar pontos específicos na história de um repositório Git.
